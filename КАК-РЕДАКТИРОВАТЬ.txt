СТУДИЯ ЗВУКОТЕРАПИИ — КАК ПРАВИТЬ САЙТ
=======================================

СОСТАВ:
  index.html — сайт (один файл)
  img/       — фотографии

КАК ПОМЕНЯТЬ ТЕКСТ, ЦЕНЫ, УСЛУГИ, ОТЗЫВЫ, FAQ, СОБЫТИЯ:
  1. Откройте index.html в любом текстовом редакторе
     (Блокнот, Notepad++, VS Code).
  2. В самом верху найдите блок «РЕДАКТИРУЕМЫЙ КОНТЕНТ САЙТА».
     Всё, что вы видите на сайте, задаётся там.
  3. Меняйте текст в кавычках "...". Кавычки и запятые не трогайте.
  4. Услуги/события/отзывы/вопросы — это списки: скопируйте блок
     {...}, вставьте рядом и поменяйте текст — появится новая
     карточка. Удалите блок — карточка исчезнет.
  5. Сохраните файл и опубликуйте изменения — см. раздел
     «КАК ОПУБЛИКОВАТЬ ИЗМЕНЕНИЯ НА САЙТ» ниже.

КАК ПОМЕНЯТЬ ФОТО:
  Просто замените файл в папке img/ на новый С ТЕМ ЖЕ ИМЕНЕМ
  (например, новое фото зала сохраните как 02-studio-hall.jpg).
  Либо положите файл с новым именем и впишите его в разделе
  images в блоке контента.
  Фото лучше сжимать до ширины ~1200px (например, на squoosh.app).

=======================================================================
КАК ОПУБЛИКОВАТЬ ИЗМЕНЕНИЯ НА САЙТ (после правки index.html или фото)
=======================================================================
  Сайт лежит на GitHub Pages. Адрес сайта:  https://www.soundspirit.ru
  Репозиторий (хранилище кода):  OlegShchegolev/sound-studio-site

  После того как сохранили файл, в терминале в папке сайта выполните
  ТРИ команды:

      git add -A
      git commit -m "правки текста"
      git push

  Дальше GitHub автоматически пересобирает сайт — это занимает
  примерно 1–2 МИНУТЫ. Сразу изменения не появляются, это нормально.

  --- ЕСЛИ ИЗМЕНЕНИЯ НЕ ПОЯВИЛИСЬ (по порядку) ---

  1. ПОДОЖДИТЕ 1–2 минуты после git push и обновите страницу.

  2. СБРОСЬТЕ КЭШ БРАУЗЕРА — самая частая причина! Браузер показывает
     старую версию из памяти. Нажмите:
        Windows: Ctrl + F5       Mac: Cmd + Shift + R
     Или откройте сайт в режиме инкогнито.

  3. ПРОВЕРЬТЕ, ЧТО PUSH РЕАЛЬНО ПРОШЁЛ. После git push не должно быть
     ошибок. Если терминал пишет «Everything up-to-date» — значит,
     коммита не было (забыли git add или git commit). Проверьте:
        git status     (должно быть «nothing to commit, working tree clean»)
        git log -1     (вверху — ваш последний коммит с вашим описанием)

  4. ПРОВЕРЬТЕ ВЕТКУ. Публикуется только ветка main.
        git branch     (звёздочка должна стоять у main)
     Если вы на другой ветке — переключитесь: git checkout main

  5. ПРОВЕРЬТЕ СБОРКУ НА GITHUB. Откройте репозиторий на github.com →
     вкладка «Actions». Последняя сборка должна быть с зелёной галочкой.
     Жёлтый кружок — ещё собирается, подождите. Красный крест — ошибка.

  Если ничего не помогло — напишите разработчику, проверит за минуту.

РАСПИСАНИЕ СОБЫТИЙ (раздел «Ближайшие события»):
  Блок events в контенте. У каждого события два поля даты:
    iso  — машинная дата "ГГГГ-ММ-ДД", напр. "2026-07-03".
           ПРОШЕДШИЕ события скрываются с сайта АВТОМАТИЧЕСКИ
           на следующий день — вручную удалять не нужно.
           Если точной даты пока нет, поставьте iso: "" — событие
           («Дата уточняется») будет висеть, пока вы его не удалите.
    date — как дата показывается людям, напр. "Чт, 3 июля · 19:00".
  Добавить событие: скопируйте строку { iso:... } и впишите свои данные.
  Когда актуальных событий не останется — весь раздел скроется сам,
  устаревших дат на сайте не будет. Достаточно изредка добавлять новые.

ВАЖНО ПРО SEO (продвижение в Яндексе/Google):
  Тексты сайта теперь записаны В ДВУХ местах:
    1) в блоке «РЕДАКТИРУЕМЫЙ КОНТЕНТ» вверху (это видят посетители);
    2) прямо в HTML-разметке ниже (это читают поисковые роботы).
  Если вы меняете важный текст (заголовки, описание услуг, цены) —
  поменяйте его И там, и там, иначе в поиске останется старая версия.
  Мелкие правки (отзывы, события) можно править только вверху.
  Проще всего: после правок напишите разработчику «обнови SEO-копию» —
  он синхронизирует за минуту.
